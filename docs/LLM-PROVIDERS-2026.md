# 🧠 چند-ارائه‌دهنده‌ی هوش مصنوعی برای نوشتن اسکریپت (بدون قطعی)

## چرا این لازم بود؟
قبلاً فکتوری فقط از Gemini استفاده می‌کرد. همون مشکلی که در پروژه `elina-radman` باعث شد سهمیه‌ی عکس تموم بشه (۴۲۹ Quota Exceeded)، دقیقاً همین ریسک برای نوشتن اسکریپت هم وجود داشت.

الان `core/llm_router.py` یک زنجیره‌ی ۸ ارائه‌دهنده رو امتحان می‌کنه، یکی‌یکی، تا اولی که کار کنه. **همه‌ی این‌ها با تست زنده روی GitHub Actions (نه فقط تئوری) بررسی شدن** — چند تا نکته‌ی مهم پیدا شد که قبلاً اشتباه فرض شده بود:

## ⚠️ اصلاحیه‌ی مهم (بر اساس تست واقعی، ۴ ژوئیه ۲۰۲۶)
- **Kimi K2 روی OpenRouter دیگه رایگان نیست** — لیست مدل‌های رایگان OpenRouter مدام عوض می‌شه؛ در تست زنده هیچ مدل Kimi/DeepSeek رایگانی پیدا نشد. کد الان به‌جای یک مدل ثابت، چند مدل پایدارتر (Llama 3.3 70B، GPT-OSS 120B) رو امتحان می‌کنه و روی خطای ۴۰۴ میره سراغ بعدی.
- **DeepSeek مستقیم با خطای ۴۰۲ (موجودی ناکافی) روبرو شد** — یعنی گرنت رایگان ۵ میلیون توکنی که تبلیغ می‌شه، برای همه‌ی حساب‌ها/مناطق به‌طور خودکار فعال نمی‌شه. باید دستی حساب `platform.deepseek.com` رو چک و شارژ کنید.

## 🆚 مقایسه‌ی GapGPT و AvalAI (طبق درخواست شما)

هر دو سایت **دقیقاً روی تعرفه‌ی رسمی OpenAI/Anthropic/Google/DeepSeek** قیمت می‌ذارن (نه قیمت اختصاصی)، پس **قیمت خام تقریباً یکسانه**. تفاوت واقعی جای دیگه‌ست:

| ویژگی | GapGPT | AvalAI |
|---|---|---|
| تعداد مدل | ~۸۹ مدل | +۴۱۰ مدل از ۲۶ ارائه‌دهنده |
| اعتبار رایگان شروع | نامشخص/محدود | ✅ **۲۰,۰۰۰ تومان + دسترسی رایگان دائمی** به GPT-4o mini، DeepSeek Chat، Gemini 2.5 Flash Lite |
| حمایت پروژه اپن‌سورس | ندیده شد | ✅ تا ۲۰ میلیون ریال اعتبار رایگان با لینک گیت‌هاب |
| پرداخت | ریالی (شتاب) | ریالی (شتاب) یا UNIT (معادل تتر) |

**نتیجه: AvalAI به‌خاطر اعتبار رایگان واقعی و پوشش وسیع‌تر مدل، در `llm_router.py` قبل از GapGPT قرار گرفته** (اما GapGPT هم به‌عنوان گزینه‌ی دوم پیاده‌سازی شده، هر دو رو می‌تونید همزمان فعال کنید).

### چطور به فکتوری اضافه بشه؟
```
AVALAI_API_KEY=...       (به‌جای پرداخت دلاری OpenAI/Anthropic مستقیم)
AVALAI_MODEL=gpt-4o-mini  (پیش‌فرض؛ می‌تونید به هر مدلی از ۴۱۰ تای موجود تغییر بدید)

GAPGPT_API_KEY=...
GAPGPT_MODEL=gpt-4o-mini
```

## 💰 تخمین واقعی مصرف توکن و هزینه‌ی ماهانه

هر ویدیو ۳ فراخوانی هوش مصنوعی داره (تحلیل رقبا + نوشتن اسکریپت + انتخاب لحظات Short):

| مرحله | توکن ورودی (تقریبی) | توکن خروجی (تقریبی) |
|---|---|---|
| تحلیل رقبا | ~۳۰۰ | ~۱۵۰ |
| نوشتن اسکریپت (۸ دقیقه) | ~۵۰۰ | ~۱۵۰۰ |
| انتخاب لحظات Short | ~۵۰۰ | ~۲۰۰ |
| **جمع هر ویدیو** | **~۱۳۰۰** | **~۱۸۵۰** |

### سناریوی واقعی: ۲ کانال، هرکدوم هفتگی (۸ ویدیو در ماه)
- مصرف کل: ۸ × ۳۱۵۰ توکن = **~۲۵,۲۰۰ توکن در ماه**
- هزینه با مدل ارزون (`gpt-4o-mini` یا `deepseek-chat`, ~$۰.۱۵-۰.۶۰ در هر میلیون توکن): **کمتر از ۱ سنت دلار در ماه**

### بدترین حالت واقع‌بینانه: ۴ کانال روزانه (۱۲۰ ویدیو در ماه)
- مصرف: ۱۲۰ × ۳۱۵۰ = ~۳۷۸,۰۰۰ توکن
- هزینه: **همچنان کمتر از ۱ دلار در ماه**

**نتیجه**: چون هر تماس کوتاهه (نه یک چت طولانی با تاریخچه)، هزینه‌ی هوش مصنوعی این پروژه ذاتاً خیلی کمه. ریسک اصلی همیشه «اشتباه کد / حلقه‌ی بی‌نهایت / تست زیاد» بوده، نه مصرف طبیعی روزمره.

## 🛡 سیستم جلوگیری از مصرف بیش‌ازحد (تازه اضافه‌شده)

طبق درخواست صریح شما (بعد از این‌که تست‌های من سهمیه‌ی روزانه رو در چند ثانیه تموم کرد)، ماژول جدید `core/usage_guard.py` ساخته شد:

- قبل از **هر** تماس با ارائه‌دهنده‌ی پولی (AvalAI، GapGPT، DeepSeek، Moonshot مستقیم)، هزینه‌ی تخمینی محاسبه می‌شه
- اگه از سقف روزانه یا ماهانه بیشتر بشه، **درخواست اصلاً فرستاده نمی‌شه** — می‌ره سراغ ارائه‌دهنده‌ی بعدی یا قالب آفلاین
- سقف‌ها با این دو متغیر قابل تنظیمن (در GitHub Secrets):
  - `LLM_DAILY_BUDGET_USD` (پیش‌فرض: $۰.۵۰)
  - `LLM_MONTHLY_BUDGET_USD` (پیش‌فرض: $۵.۰۰)
- یک لجر (`channels/usage_ledger.json`) مصرف واقعی روز/ماه رو ثبت می‌کنه — قابل مشاهده در گیت‌هاب

## 🐛 باگ دیگری که در همین بررسی پیدا و رفع شد: `upload_frequency` اجرا نمی‌شد
هنگام محاسبه‌ی هزینه‌ی ماهانه، متوجه شدم فیلد `upload_frequency` (که موقع ثبت کانال تعیین می‌کنید — روزانه/هفتگی/دوهفته‌ای/ماهانه) **هیچ‌وقت واقعاً چک نمی‌شد** — کرون روزانه، برای هر کانال فعال، هر روز یک ویدیوی کامل می‌ساخت، صرف‌نظر از این تنظیم! این می‌تونست هزینه رو (در مقیاس بزرگ‌تر) تا ۷ برابر بیشتر از انتظار کنه. ماژول جدید `core/schedule_guard.py` این رو کاملاً درست کرد — از این به بعد اگه کانالی رو `weekly` ثبت کنید، واقعاً فقط یک‌بار در هفته اجرا می‌شه.

## 🛍 بررسی سایت‌های فروش اکانت آماده (لایسنس مارکت و مشابه)

طبق درخواستتون، این گزینه رو هم بررسی کردم: به‌جای API pay-as-you-go، خرید یک **اکانت اشتراکی آماده** (مثلاً ChatGPT Plus یا Claude Pro) از سایت‌های ایرانی مثل `license-market.ir` یا `20payment.com`.

### قیمت‌های یافت‌شده (خرید مستقیم اشتراک، نه API)
| سرویس | قیمت تقریبی (تومان) | نوع |
|---|---|---|
| ChatGPT Plus (اختصاصی) | ~۱,۲۰۰,۰۰۰ - ۱,۵۰۰,۰۰۰ | ماهانه |
| Claude Pro | ~۳,۵۶۰,۴۰۰ - ۱۰,۶۹۰,۰۰۰ (بسته به فروشنده) | ماهانه |

### چرا این گزینه برای پروژه‌ی ما **مناسب نیست**
این نوع اکانت‌ها برای **استفاده‌ی انسانی از طریق مرورگر/اپ چت** طراحی شدن (یعنی شما با دست تایپ می‌کنید و جواب می‌گیرید) — **هیچ‌کدوم API واقعی برای اتصال خودکار نمی‌دن**. پروژه‌ی ما نیاز به یک کلید API داره که کد پایتون بتونه خودکار (بدون دخالت انسان، هزاران بار در ماه) باهاش صحبت کنه — این قابلیت **فقط در پلن‌های API pay-as-you-go** (مثل AvalAI/GapGPT/DeepSeek مستقیم) وجود داره، نه در اشتراک‌های چت معمولی.

**نتیجه**: خرید اشتراک آماده هم گرون‌تره (۱.۲-۱۰ میلیون تومان ثابت در ماه) هم اصلاً کار نمی‌کنه چون API نمی‌ده. با توجه به این‌که طبق محاسبه‌ی بالا، هزینه‌ی واقعی API برای این پروژه **کمتر از ۱ دلار در ماه** هست، حتی اگه اشتراک آماده API می‌داد هم به‌صرفه نبود. **پیشنهاد قطعی: AvalAI یا GapGPT با پرداخت pay-as-you-go، نه خرید اشتراک آماده.**
